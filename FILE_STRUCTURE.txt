FILE DIRECTORY AND ARCHITECTURE GUIDE
================================================================================
This document outlines the purpose of every file and directory in the project workspace for the Precision-Aware Type Demotion Framework.

CORE COMPILER SOURCE
--------------------------------------------------------------------------------
src/tool.cpp
This is the core C++ source code for the Clang compiler plugin. It contains all the logic for Deliverables 1, 2, and 3. Because this is the most important file in the project, here is a detailed breakdown of its architecture:

  1. Data-Flow Graph (DFG) Struct:
     - Defines the 'DFGNode' structure which stores variable names, current types, mathematical operations, and error budgets.
     - Maintains a global memory dictionary ('Graph') linking variable names to their mathematical dependencies.

  2. AssignmentCallback (The AST Analyzer - Pass 1):
     - Uses a 'StatementMatcher' to hunt for mathematical assignments (e.g., c = a + b).
     - It uses 'ignoringParenImpCasts' to safely look through invisible C compiler casts.
     - When it finds an equation, it parses the right-hand side using a nested 'RecursiveASTVisitor' to extract the operator (+, *), constants (like 2.0), and variable dependencies.

  3. propagateError() (The Math Engine):
     - Calculates the "Backward Precision Propagation".
     - Starts at the final output variable with an initial error budget (e.g., 0.05).
     - Recursively walks backward through the graph. For addition, error budgets transfer directly. For multiplication, the error budget is shrunk by dividing by the multiplier.
     - Flags variables as "safe" if their remaining budget is larger than a 16-bit float's natural epsilon limit (0.001).

  4. RewriteCallback (The AST Rewriter - Pass 2):
     - Runs a second pass over the AST specifically looking for Variable Declarations ('VarDecl').
     - Checks the graph to see if the variable was mathematically proven safe for demotion.
     - If safe, it uses the 'clang::Rewriter' module to physically change the word "float" to "_Float16" in the source code.
     - It uses a 'std::set' memory tracker to prevent duplicating the "_Float16" keyword when variables are declared on the same line (like "float a, b;").
     - Finally, it saves the optimized code to 'test_demoted.c'.

CMakeLists.txt
The build configuration file. It tells the compiler how to link our custom tool.cpp against the massive LLVM and Clang development libraries.


BUILD & VERIFICATION
--------------------------------------------------------------------------------
build/
The directory where all compiled executables and scripts are stored to keep the main workspace clean.

build/verify.py
The Python automation script used for Deliverable 4. It takes a C file, compiles it normally (FP32), runs the Clang tool to optimize it, compiles the new optimized code (FP16), and then runs both to verify that the absolute mathematical error stays within the 0.05 limit.


TESTING & EVALUATION FILES
--------------------------------------------------------------------------------
test.c
The initial, simple test file we used to build the compiler. It contains a basic mathematical chain (a = 10, b = a*2, c = b+5).

test_demoted.c
This is an auto-generated file. Whenever the compiler tool runs on a file (like test.c), it outputs the optimized version here, with the safe "float" variables converted to "_Float16".

kernels/
This directory contains the 5 evaluation files required for Deliverable 5. Each file simulates a real-world mathematical application:

1. kernels/kernel_dot_product.c
Simulates a dot-product operation commonly used in Neural Network matrix multiplications.

2. kernels/kernel_fir_filter.c
Simulates a Digital Signal Processing (DSP) filter accumulating signals based on coefficients.

3. kernels/kernel_softmax.c
Simulates the Softmax probability function used at the end of AI classification models (using Taylor series approximations).

4. kernels/kernel_sigmoid.c
Simulates a Sigmoid activation layer for neural networks.

5. kernels/kernel_polynomial.c
Simulates a complex polynomial mathematical approximation (y = 2x^3 - 1.5x^2 + 0.5x + 1).
